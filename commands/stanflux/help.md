# STAN.FLUX Hilfe

Zeige verfügbare STAN.FLUX Commands und wie sie zu nutzen sind.

## Verfügbare Commands

| Command | Beschreibung |
|---------|-------------|
| `/stanflux:retro` | Retrospektive - Learnings identifizieren, Regeln verbessern |
| `/stanflux:challenge` | Idee/Entscheidung aus 6 Perspektiven herausfordern |
| `/stanflux:help` | Diese Hilfe anzeigen |

## STAN.FLUX Kurzübersicht

### Kern-Prinzipien
- **Wissen erst:** Wissen sichern VOR handeln
- **Sicherheits-Warnung:** Sich sicher fühlen = Warnsignal
- **Ehrlichkeit:** "Ich weiß es nicht" ist OK

### Wichtige Regeln
- **Brownfield:** Verstehen VOR Ändern
- **Vor dem Bauen:** Ziel klären VOR Lösung
- **Perspektiven:** 6 Perspektiven bei nicht-trivialen Entscheidungen
- **Empfehlung-Pflicht:** Position beziehen, nicht nur Optionen
- **Validierung:** Echte Prüfung VOR done
- **Eskalation:** Risiken dokumentieren + User bestätigt

### Bei Problemen
- **3-Strikes:** Bei ≥3 gleichen Fehlern → STOPP + Perspektivwechsel
- **Multi-Krise:** Bei Überlastung → STOPP + sortieren

## Regeln anpassen

Die Regeln befinden sich in `stanflux.md`. Änderungen sollten über `/stanflux:retro` erfolgen, um sicherzustellen dass sie aus echten Patterns entstehen.
